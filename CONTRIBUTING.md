# Contributing

Thanks for contributing to this project.

## Development Setup

1. Fork the repository.
2. Clone your fork locally.
3. Open `index.html` in a browser to test changes.
4. Keep changes focused and easy to review.

## Contribution Guidelines

- Use plain HTML/CSS/JavaScript (no framework dependency).
- Preserve existing app behavior unless the PR explicitly changes it.
- Keep UI responsive for desktop and mobile.
- Test manually before opening a pull request:
  - Add, edit, delete task
  - Add, rename, delete column
  - Drag-drop between columns and reorder inside a column
  - Search and priority filter
  - Theme toggle
  - Export and import JSON backup

## Commit Style

- Use clear, descriptive commit messages.
- Keep one logical change per commit when possible.

## Pull Requests

Please include:

- What changed
- Why it changed
- Screenshots/GIFs for UI updates (if applicable)
- Manual test notes

## Reporting Issues

When reporting a bug, include:

- Browser name/version
- Steps to reproduce
- Expected result
- Actual result
- Screenshot/recording (if possible)
